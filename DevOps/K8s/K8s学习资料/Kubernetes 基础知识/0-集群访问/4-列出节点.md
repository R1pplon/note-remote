# 列出节点

在此步骤中，你将检查集群中的节点。节点是 Kubernetes 中的工作机器（虚拟或物理）。由于我们使用的是 Minikube，我们期望看到一个既充当控制平面又充当工作节点的单一节点。

在较大的集群中，这些角色通常是分开的。控制平面节点专注于编排，而工作节点运行应用程序负载。Minikube 将它们合并为一个节点以简化学习环境，但命令输出仍然展示了你在生产环境中会看到的相同概念。

要列出集群中的所有节点，请使用 `get nodes` 命令：

```bash
kubectl get nodes
```

你将看到类似以下的输出：

```plaintext
NAME       STATUS   ROLES           AGE     VERSION
minikube   Ready    control-plane   5m20s   v1.26.1
```

让我们解析一下各列：

- **NAME**：节点的名称（在本例中为 `minikube`）。
- **STATUS**：状态应为 `Ready`，表示节点健康且准备好接收工作负载。
- **ROLES**：显示节点的角色（例如 `control-plane`）。
- **AGE**：节点运行的时间。
- **VERSION**：节点上安装的 Kubernetes 版本。

如果状态为 `NotReady`，通常意味着节点仍在启动中或存在网络问题。由于你的状态是 `Ready`，说明你的集群已完全正常运行。
