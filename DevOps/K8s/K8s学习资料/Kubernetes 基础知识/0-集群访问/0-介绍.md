# 简介

Kubernetes 是一个容器编排系统。它负责管理容器化应用程序在集群中的调度、运行、暴露和恢复方式。

要与 Kubernetes 集群进行交互，通常需要使用 `kubectl`。在部署应用程序之前，你需要一个正在运行的集群以及一个指向该集群的有效客户端配置。

在本实验中，你将完成设置 Kubernetes 集群访问权限的基本步骤。你将使用 **Minikube**，这是一个在本地运行单节点 Kubernetes 集群的工具，非常适合学习和开发。

你将学习如何：

1. 启动本地 Kubernetes 集群。
2. 验证 `kubectl` 上下文，确保目标集群正确。
3. 获取必要的集群信息。
4. 列出集群中的节点以验证其状态。
