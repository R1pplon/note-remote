# 检查集群信息

在此步骤中，你将获取有关集群的基本信息，以验证连接性并查看 Kubernetes 控制平面的运行位置。

`cluster-info` 命令是检查 `kubectl` 是否能成功与 API 服务器通信的简单方法。

Kubernetes 是一个 API 驱动的系统。几乎每一个操作，从创建 Pod 到扩展 Deployment，都会变成对控制平面的 API 请求。这就是为什么「我的客户端能否连接到 API 服务器？」是任何集群排查中的首要健康检查项。

运行以下命令：

```bash
kubectl cluster-info
```

你应该会看到类似以下的输出：

```plaintext
Kubernetes control plane is running at https://127.0.0.1:32769
CoreDNS is running at https://127.0.0.1:32769/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy

To further debug and diagnose cluster problems, use 'kubectl cluster-info dump'.
```

IP 地址和端口可能会有所不同，但关键点在于 "Kubernetes control plane is running"（Kubernetes 控制平面正在运行）。这确认了你的客户端 (`kubectl`) 已成功连接到服务器（集群）。
