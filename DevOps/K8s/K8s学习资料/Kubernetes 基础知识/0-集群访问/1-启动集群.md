# 启动集群

在此步骤中，你将使用 Minikube 启动一个本地 Kubernetes 集群。Minikube 会在你的本地机器上创建一个虚拟机或容器，并在其中部署一个简单的集群。这使你无需云服务提供商即可练习 Kubernetes 命令。

在这一阶段，明确区分三个概念很有帮助。**集群 (Cluster)** 是整个 Kubernetes 环境。**节点 (Node)** 是集群中的一台机器。**控制平面 (Control plane)** 是接收 API 请求并决定执行什么操作的部分。Minikube 将这些组件打包成一个本地学习环境，让你在接触云基础设施之前，能够专注于 Kubernetes 的核心概念。

首先，确保你位于项目目录中：

```bash
cd ~/project
```

现在，启动 Minikube 集群。我们将指定 `docker` 驱动程序，以便在 Docker 容器内运行集群，这与本实验环境兼容。

```bash
minikube start --driver=docker
```

此过程可能需要几分钟，因为它会下载必要的 Kubernetes 镜像并启动控制平面。你将看到 Minikube 正在拉取镜像、启动容器并配置集群的输出信息。

命令完成后，你可以检查 Minikube 的状态以确保其运行正常：

```bash
minikube status
```

你应该会看到输出显示 `host`、`kubelet` 和 `apiserver` 均为 "Running"（运行中）。
