# 验证上下文

在此步骤中，你将验证 `kubectl` 命令行工具是否已配置为与你的新 Minikube 集群进行通信。

Kubernetes 使用配置文件（通常位于 `~/.kube/config`）来存储集群访问信息。此文件可以包含多个集群的详细信息。「上下文 (Context)」决定了 `kubectl` 将命令发送到哪个集群。当你在上一步启动 Minikube 时，它会自动更新你的配置并将上下文设置为 `minikube`。

理解这个概念很重要，因为 `kubectl` 只是一个客户端。它并不「包含」集群。它读取连接详细信息、证书和当前选定的上下文，然后向 Kubernetes API 服务器发送请求。在实际工作中，工程师经常需要在开发、测试和生产集群之间切换，因此检查当前上下文是一种安全习惯，而不仅仅是形式。

要验证当前上下文，请运行以下命令：

```bash
kubectl config current-context
```

输出应为：

```plaintext
minikube
```

如果你想查看完整的配置详细信息，包括集群服务器 URL 和用户证书（已安全隐藏），可以查看配置：

```bash
kubectl config view
```

这确认了你后续运行的任何 `kubectl` 命令都将在你的本地 Minikube 集群上执行。
