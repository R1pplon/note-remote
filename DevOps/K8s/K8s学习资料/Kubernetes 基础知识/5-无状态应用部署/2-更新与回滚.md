# 更新与回滚

开发团队推送了一个新功能，但用户反馈出现了问题。你需要执行滚动更新以部署新版本，验证发布状态，然后立即回滚到稳定版本以恢复服务质量。

## 任务

1. 将 `webapp` Deployment 更新为使用镜像 `nginx:1.19.1`（模拟新版本）。
2. 等待滚动更新完成。
3. **立即回滚** Deployment 到上一个版本（恢复为 `nginx:1.19.0`）。

## 要求

- 首先执行更新至 `nginx:1.19.1`。
- Deployment 的最终状态必须使用稳定镜像 `nginx:1.19.0`。
- 你必须演示如何使用滚动更新管理命令。

## 示例

要查看滚动更新的历史记录，可以使用：

```bash
kubectl rollout history deployment/webapp
```

## 提示

<details>
<summary>更新镜像</summary>

使用 `kubectl set image deployment/webapp <container-name>=nginx:1.19.1`。默认的容器名称通常是 `nginx`。

</details>

<details>
<summary>查看状态</summary>

使用 `kubectl rollout status deployment/webapp` 来观察进度。

</details>

<details>
<summary>回滚</summary>

使用 `kubectl rollout undo deployment/webapp` 来恢复到上一个修订版本。

</details>
