# 部署与暴露

你的首要任务是在集群中建立该应用。你需要创建一个 Deployment 资源来管理应用实例，并创建一个 Service 将其暴露到网络中。

## 任务

1. 创建一个名为 `webapp` 的 Deployment，使用镜像 `nginx:1.19.0`，并设置 **3 个副本**。
2. 使用名为 `webapp-svc` 的 Service 暴露该 Deployment，类型为 `NodePort`，端口为 **80**。

## 要求

- **Deployment 名称**: `webapp`
- **镜像**: `nginx:1.19.0`
- **副本数**: 3
- **Service 名称**: `webapp-svc`
- **Service 端口**: 80
- **Service 类型**: `NodePort`

## 示例

你可以使用以下命令验证资源状态：

```bash
kubectl get deployment webapp
kubectl get svc webapp-svc
```

输出应显示 3/3 个就绪的副本，且 Service 已暴露 80 端口。

## 提示

<details>
<summary>通过命令行部署</summary>

你可以使用 `kubectl create deployment` 命令来创建 Deployment。

</details>

<details>
<summary>扩缩容</summary>

使用 `kubectl scale deployment webapp --replicas=3`。

</details>

<details>
<summary>通过命令行暴露服务</summary>

使用 `kubectl expose deployment webapp ...` 可以快速创建 Service。

</details>
