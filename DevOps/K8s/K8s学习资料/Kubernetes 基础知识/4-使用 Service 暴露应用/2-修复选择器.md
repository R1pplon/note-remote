# 修复选择器

Service 使用 **Selector**（选择器）来查找它应该将流量路由到的 Pod。它就像一个邮寄标签。如果信封上的地址（Service Selector）与房子上的地址（Pod Label）不匹配，邮件就永远无法送达。

标签和选择器在 Kubernetes 中随处可见，不仅仅是在 Service 中。它们是该平台核心的分组机制之一。Kubernetes 资源通常通过匹配标签来查找相关对象，而不是硬编码具体的 Pod 名称，这使得系统在 Pod 被重新创建时保持了灵活性。

在设置步骤中，我们已经为你自动启动了一个 Web 应用。让我们找到它的“地址标签”。

运行以下命令查看正在运行的 Pod 的标签：

```bash
kubectl get pods --show-labels
```

你应该会看到类似这样的内容：`web-app-xxxxx ... app=web-app`

标签是 `app=web-app`。然而，我们的 Service 正在寻找 `app: unknown`。让我们修复它。

编辑你的 `service.yaml` 文件：

```bash
nano service.yaml
```

将 `selector` 部分修改为与你的 Pod 标签匹配：

```yaml
selector:
  app: web-app
```

保存并退出。现在 Service 准确地知道该把流量发送到哪里了。
