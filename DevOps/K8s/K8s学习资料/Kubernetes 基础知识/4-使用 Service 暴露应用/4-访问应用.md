# 访问应用

现在是见证奇迹的时刻。我们能从“外部”（你的终端）访问该应用吗？

该应用已在 Kubernetes 节点的 `30008` 端口上暴露。

这里涉及三个端口，初学者经常会混淆它们。**nodePort** 是节点上对外开放的端口，**port** 是 Service 抽象暴露的端口，而 **targetPort** 是容器内部实际提供服务的端口。明确这些角色的区别将防止以后产生困惑。

首先，让我们获取 Minikube 节点的 IP 地址：

```bash
minikube ip
```

假设它是 `192.168.49.2`。你应用的完整地址就是 `http://192.168.49.2:30008`。

我们可以使用 `curl` 访问此地址并验证应用是否在响应：

```bash
curl http://$(minikube ip):30008
```

你应该能看到“Welcome to nginx!”页面的 HTML 代码。

这确认了请求的完整路径：

1. 你调用了节点的外部端口（`30008`）。
2. **Service** 接听了呼叫。
3. 它检查了自己的 **Selector**（`app: web-app`）。
4. 它将呼叫转发给匹配该标签的其中一个 **Pod**。
5. Pod 返回了网页。
