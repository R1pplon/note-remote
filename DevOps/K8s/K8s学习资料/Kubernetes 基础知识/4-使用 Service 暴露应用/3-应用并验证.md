# 应用并验证

现在地址已经正确，让我们启动 Service。

当你应用此清单时，Kubernetes 会创建一个 Service 对象并配置内部网络规则，以便到达 Service 的请求可以被转发到匹配的 Pod 端点。需要理解的重要一点是，Service 本身并不运行你的应用。它只是应用 Pod 前端的一个网络抽象。

将配置应用到集群：

```bash
kubectl apply -f service.yaml
```

你应该会看到：`service/my-service created`。

要确认 Service 已启动并运行，请检查服务列表：

```bash
kubectl get svc my-service
```

你将看到：

- **NAME**: my-service
- **TYPE**: NodePort
- **PORT(S)**: 80:30008/TCP

这确认了你的“电话号码”已激活，并在 30008 端口等待呼叫。
