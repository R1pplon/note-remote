# 扩缩容

想象一下你的网站变得很受欢迎。3 台 Web 服务器已经不够用了，你需要 5 台。你需要手动创建新文件或匆忙创建 Pod 吗？

扩展 Deployment 会改变声明的期望数量，而不仅仅是当前运行的进程集。这就是为什么新的规模在一次手动操作之后依然有效。控制器会继续将更新后的副本数视为真理，直到你再次更改它。

不需要。你只需要告诉管理器增加数量即可。

我们可以使用 `kubectl scale` 命令动态更新副本数量。

将副本数量从 3 更改为 5：

```bash
kubectl scale deployment/nginx-deployment --replicas=5
```

现在，见证奇迹发生：

```bash
kubectl get pods
```

你现在会看到 **5** 个 Pod 在运行。为了满足新的要求，系统启动了两个新的 Pod。

你也可以进行缩容。如果流量下降，你可以通过减少数量来节省资源：

```bash
kubectl scale deployment/nginx-deployment --replicas=2
```

再次运行 `kubectl get pods`，你会看到 Kubernetes 正在终止多余的 Pod，直到只剩下 2 个。

这种动态扩缩容正是企业喜爱 Kubernetes 的原因。
