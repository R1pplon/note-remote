# 应用并观察

现在你已经有了工作说明书，让我们雇佣这位管理器。将配置应用到集群中：

当你创建 Deployment 时，Kubernetes 通常会在后台创建另一个名为 **ReplicaSet** 的对象。ReplicaSet 是直接负责保持所需数量 Pod 存活的组件。初学者暂时不需要直接管理 ReplicaSet，但了解它们的存在很有用，因为你经常会在命令输出中看到它们。

```bash
kubectl apply -f deployment.yaml
```

Kubernetes 将确认：`deployment.apps/nginx-deployment created`。

Deployment（管理器）将立即开始创建 3 个 Pod。要快速查看它们，请运行：

```bash
kubectl get pods
```

你应该会看到 3 个不同的 Pod，名称均以 `nginx-deployment-` 开头：

```plaintext
NAME                                READY   STATUS    RESTARTS   AGE
nginx-deployment-66b6c48dd5-8j64w   1/1     Running   0          5s
nginx-deployment-66b6c48dd5-d95l2   1/1     Running   0          5s
nginx-deployment-66b6c48dd5-lzr5w   1/1     Running   0          5s
```

你也可以检查 Deployment 本身的状态：

```bash
kubectl get deployments
```

它应该显示 `READY 3/3`，确认所有 3 个请求的副本都已启动并运行。
