# 简介

在之前的实验中，你创建了单个 Pod。虽然 Pod 是最基本的构建单元，但手动管理它们非常繁琐。如果一个 Pod 挂了，你必须手动重启它；如果你想要更多副本，也必须手动创建更多 Pod。

这时就需要用到 **Deployment**。

Deployment 是一个更高级的控制器，它为你管理副本集（ReplicaSet）和 Pod。你只需声明需要多少个副本以及它们应该使用什么 Pod 模板，Kubernetes 就会努力让实际状态与你的声明保持一致。

这里的关键转变是从管理单个 Pod 转变为管理可重复的工作负载。单个 Pod 是脆弱的，因为它只是一个实例。而 Deployment 描述了创建 Pod 的模式以及应该存在的副本数量目标。正是这种模式使得扩缩容和自我修复成为可能。

在本实验中，你将学习：

- 创建一个具有特定目标（3 个副本）的严格管理器（Deployment）。
- 观察管理器如何雇佣工作者（Pod）。
- “干掉”一个工作者来测试管理器的反应（自我修复）。
- 要求管理器雇佣更多工作者（扩缩容）。
