# 查看日志

在这一步中，你将学习如何检查运行中 Pod 的日志。可以将日志视为应用程序的「声音」。如果出现问题，日志通常是寻找线索的首选之地。

在底层，`kubectl logs` 通常会读取主容器进程写入标准输出（standard output）和标准错误（standard error）的内容。这就是为什么许多容器化应用程序被设计为将日志输出到终端，而不是写入本地文件。在 Kubernetes 中，平台更容易收集并统一展示终端输出。

我们已经在后台为你启动了一个名为 `my-nginx` 的 Pod。首先，验证它是否正在运行：

```bash
kubectl get pods
```

你应该能看到 `my-nginx` 的状态为 `Running`。如果它尚未就绪，请稍等片刻再试。

一旦 Pod 运行起来，使用 `kubectl logs` 命令读取容器输出到标准输出的内容：

```bash
kubectl logs my-nginx
```

由于 Nginx 是一个稳定的 Web 服务器，它最初可能比较安静，输出很少，或者只显示一条准备好连接的消息。

要实时查看日志（就像观看直播一样），你可以添加 `-f` (follow) 参数：

```bash
kubectl logs -f my-nginx
```

你会看到光标挂起，等待新的消息。这对于实时观察流量非常有用。按 `Ctrl+C` 停止跟踪日志并返回终端提示符。
