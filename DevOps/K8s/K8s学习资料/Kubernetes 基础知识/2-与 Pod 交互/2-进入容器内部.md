# 进入容器内部

有时仅阅读日志是不够的。你可能需要检查配置文件是否存在、验证是否安装了某个工具，或者从 Pod *内部*测试网络连通性。

你可以使用 `kubectl exec` 命令来完成这些操作。它允许你在运行中的容器内执行命令。我们通常用它来打开一个交互式 Shell，从而实现「瞬间移动」到容器内部。

这里有一个重要的区别：`kubectl exec` 并不像 SSH 那样将你「移动」到另一台机器。它只是要求节点上的 kubelet 在已运行的容器内启动一个新进程。这就是为什么你看到的是容器的文件系统和环境，而不是你本地终端的环境。

运行以下命令在 `my-nginx` Pod 内打开一个 `bash` Shell：

```bash
kubectl exec -it my-nginx -- /bin/bash
```

让我们理解一下这些参数：

- `-i` (Interactive)：保持输入打开，以便你可以输入内容。
- `-t` (TTY)：分配一个伪终端，使其看起来像一个正常的命令行提示符。
- `--`：告诉 kubectl「此后的所有内容都是发给容器的命令，而不是发给 kubectl 的」。

你的命令提示符应该会变成类似 `root@my-nginx:/#` 的样子。现在你已经进入了 Kubernetes 集群中特定的容器内部！

尝试列出目录中的文件，确认你处于一个新的环境中：

```bash
ls -F
```

你现在看到的是 Nginx 容器内的文件，它们与你的 LabEx 工作区文件完全隔离。
