# 检查环境

既然你已经「进入」了容器，让我们四处看看。一个常见的调试任务是检查容器运行的操作系统 (OS)。这有助于你了解哪些命令（如 `apt` 或 `apk`）可用。

这也引出了一个关于容器的常见课题：容器镜像决定了你可用的用户空间工具和文件。两个 Pod 可能都是「Linux」，但差异巨大，因为一个镜像基于 Debian，另一个基于 Alpine，而有的镜像甚至几乎不包含任何 Shell 工具。

在提示符仍为 `root@my-nginx:/#` 时，运行此命令读取 OS 发行版文件：

```bash
cat /etc/os-release
```

你将看到标识 Linux 发行版的输出，很可能是「Debian GNU/Linux」。这是因为官方 Nginx 镜像是在 Debian 之上构建的。

探索完成后，你需要退出容器并返回主终端。输入：

```bash
exit
```

你的提示符将返回到 `labex:~/project`。

为了确认你完成了此项调查，创建一个名为 `os_version.txt` 的文件，并将你发现的操作系统名称（例如「Debian」）写入其中。

```bash
echo "Debian" > ~/project/os_version.txt
```
