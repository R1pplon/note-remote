# 端口转发

默认情况下，Pod 是隔离的。它们就像封闭社区中的房子——它们可以互相交谈，但外人（比如你在本地终端）无法轻易触及它们。

为了从本地机器访问 `my-nginx` 中运行的 Web 服务器，我们可以使用**端口转发 (Port Forwarding)** 创建一条「秘密通道」。

端口转发是一种调试便利手段，而不是生产环境中发布应用程序的标准方式。它会在你的当前工作站和一个 Pod 之间建立一条临时通道。这非常适合检查和快速测试，因为它避免了创建完整的 Service 或更改外部网络策略。

运行以下命令，将本地端口 `8080` 转发到 Pod 的端口 `80`：

```bash
kubectl port-forward my-nginx 8080:80 &
```

**注意末尾的 `&`！** 这告诉 Linux 在后台运行此进程。如果你不加这个符号，该命令会挂起你的终端，在你停止它之前，你将无法输入任何其他内容。

你应该会看到一条消息，显示 `Forwarding from 127.0.0.1:8080 -> 80`。

现在，你可以使用 `curl` 通过这条通道访问 Nginx 网页：

```bash
curl localhost:8080
```

你应该能看到「Welcome to nginx!」页面的 HTML 代码。成功了！你已经通过通道深入集群并获取了数据。
