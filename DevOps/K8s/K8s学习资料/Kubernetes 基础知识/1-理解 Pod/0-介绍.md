# 简介

在本实验中，你将探索 Kubernetes 中最小的可部署单元：**Pod**。

Pod 是 Kubernetes 调度到节点（Node）上的对象。在 Kubernetes 中，你通常不会直接运行容器，而是将容器放置在 Pod 中。

一个 Pod 可以包含一个或多个共享网络和存储上下文的容器。大多数初学者示例使用单个容器，但共享环境的设计理念才是其核心所在。

你将学习如何：

- 使用 YAML 配置文件（蓝图）定义 Pod。
- 使用 `kubectl` 在集群中创建 Pod。
- 检查 Pod 的状态以确保其正在运行。
- 查看 Pod 的详细配置和历史记录。

实验结束时，你将在集群中的 Pod 内运行一个真实的 Web 服务器。
