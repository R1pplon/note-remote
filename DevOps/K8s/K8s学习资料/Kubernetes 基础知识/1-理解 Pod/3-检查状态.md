# 检查状态

在这一步中，你将检查新 Pod 的进度。创建 Pod 并非瞬间完成；下载容器镜像并启动应用程序需要一点时间。

记住这一点很有用：`kubectl apply` 通常会很快返回，因为它只是提交了请求。实际工作是在集群内部异步进行的。这就是为什么检查状态是正常的第二步，而不是因为出了问题。

要查看 Pod 的状态，请使用 `get` 命令：

```bash
kubectl get pods
```

你将看到一个列出 Pod 的表格。查看 `STATUS` 列：

```plaintext
NAME        READY   STATUS    RESTARTS   AGE
nginx-pod   1/1     Running   0          15s
```

- **STATUS**：这是最重要的字段。
  - `ContainerCreating`：系统仍在下载镜像或启动容器。
  - `Running`：成功！Pod 已启动且运行正常。
  - `Pending`：集群正在计算将 Pod 放置在何处。
  - `Error` 或 `CrashLoopBackOff`：出错了（例如，应用程序崩溃）。

- **READY**：显示 `1/1`，意味着 Pod 中定义的 1 个容器中有 1 个已准备好处理流量。

如果你看到 `ContainerCreating`，请稍等几秒钟，然后再次运行 `kubectl get pods`。一旦显示为 `Running`，你的 Nginx Web 服务器就上线了。
