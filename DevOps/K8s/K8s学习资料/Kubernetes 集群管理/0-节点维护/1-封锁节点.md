# 封锁节点

在进行任何维护之前，我们必须确保 Kubernetes 调度器不会将任何「新」的 Pod 放置在我们即将维护的节点上。这个过程称为「封锁」（cordoning）。这就像在酒店房间门上挂上「请勿打扰」或「暂停服务」的牌子。

这是一个调度决策，而不是关机。被封锁的节点仍然可以保持健康，并继续运行其现有的 Pod。Kubernetes 只是将其标记为不可用于新的放置，以便你可以以受控的方式开始维护。

首先，让我们确定节点的名称。运行以下命令：

```bash
kubectl get nodes
```

你应该能看到列出的一个节点（例如 `minikube`），其状态为 `Ready`。

要将节点标记为不可调度，请使用 `cordon` 命令。我们将使用命令替换 `$()` 来自动获取节点名称：

```bash
kubectl cordon $(kubectl get nodes -o jsonpath='{.items[0].metadata.name}')
```

现在，再次验证节点的状态：

```bash
kubectl get nodes
```

你应该会看到状态变为 `Ready,SchedulingDisabled`。这确认了节点现在已被封锁。不会有新的 Pod 被调度到这里，但现有的 Pod 仍在正常运行。
