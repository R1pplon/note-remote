# 验证状态

驱逐节点后，验证应用程序的情况非常重要。

这一验证步骤教会了集群管理员一个重要的习惯：不要在发出维护命令后就停止操作。始终从工作负载的角度检查调度结果。只有当应用程序落在它们应该落的地方时，维护操作才算成功。

检查 Pod 的状态：

```bash
kubectl get pods
```

如前所述，你很可能会看到 Pod 处于 `Pending` 状态。

为了了解它们为何处于挂起状态，让我们询问 Kubernetes。描述其中一个 Pod：

```bash
kubectl describe pod $(kubectl get pods -o jsonpath='{.items[0].metadata.name}')
```

查看底部的 **Events** 部分。你应该会看到类似「0/1 nodes are available: 1 node(s) are unschedulable」的消息。

这确认了驱逐是成功的。调度器想要运行该 Pod，但它尊重我们在唯一可用节点上放置的「请勿打扰」标志。
