# 解除封锁节点

想象一下，你现在已经完成了「翻新」（操作系统升级或维护）。现在是时候重新开放房间并将节点恢复服务了。

`uncordon` 只是移除了调度限制。它不会立即强制将 Pod 放入节点。它告诉调度器该节点再次有资格进行未来的放置，然后正常的协调机制就会接管。

为此，我们使用 `uncordon` 命令。这将移除「请勿打扰」的牌子。

```bash
kubectl uncordon $(kubectl get nodes -o jsonpath='{.items[0].metadata.name}')
```

现在，验证节点状态：

```bash
kubectl get nodes
```

状态应该恢复为 `Ready`（`SchedulingDisabled` 标签已消失）。

最后，再次检查你的 Pod。由于节点已重新开放，Kubernetes 调度器现在应该将那些 `Pending` 的 Pod 放回该节点。

```bash
kubectl get pods
```

你应该会看到它们从 `Pending` 转换为 `ContainerCreating`，最后变为 `Running`。
