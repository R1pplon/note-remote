# 简介

在生产环境的 Kubernetes 集群中，节点经常需要进行维护。挑战在于如何在不造成工作负载不必要中断的情况下完成这些操作。

可以将节点想象成「酒店房间」，而 Pod 则是「房客」。如果你需要翻新房间（执行维护），你不能在房客睡觉时强行闯入。你需要一个礼貌且安全的过程。

在本实验中，你将学习节点维护的标准工作流程：

- **Cordon（请勿打扰）**：在门上挂上「不接待新房客」的牌子。酒店（调度器）将停止向该房间安排新房客。
- **Drain（驱逐）**：礼貌地请现有房客离开，并将他们转移到其他可用房间。
- **Maintenance（维护）**：执行必要的工作（翻新）。
- **Uncordon（开放）**：取下牌子，允许再次进行调度。

你将使用本地的 Minikube 集群来模拟这一过程。
