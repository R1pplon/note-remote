# 简介

在软件开发中，将数据库 URL 或密码等设置硬编码到应用程序代码中是一种糟糕的做法。这不仅会增加变更的难度，还会带来安全风险。

Kubernetes 通过以下两种对象解决了这个问题：

1. **ConfigMap**：可以将其想象成一个**公共公告板**。你可以在这里张贴信息（例如「背景颜色为蓝色」或「数据库地址为 10.0.0.1」）。办公室里的任何人（Pod）都可以读取它。
2. **Secret**：用于存储密码或令牌等敏感值。

这种应用程序代码与运行时配置的分离是平台设计的一项重要原则，而不仅仅是 Kubernetes 的一项功能。

在本实验中，你将：

- 在公告板（ConfigMap）上写一张便条。
- 将密码放入保险箱（Secret）。
- 教导 Pod 如何读取这两者。
