# 定义角色

首先，我们需要创建权限集。我们想要一个「只读」角色。该角色允许用户 `list`（列出）和 `get`（获取）Pod，但不能 `create`（创建）或 `delete`（删除）它们。

RBAC 是 Kubernetes 的授权机制，而非身份验证。身份验证回答的是「你是谁？」，而授权回答的是「你被允许做什么？」。本实验重点关注第二个问题，即通过在特定资源类型上定义明确的允许操作（称为 **verbs**，动词）来解决。

创建一个名为 `role.yaml` 的文件：

```bash
nano role.yaml
```

粘贴以下 YAML 内容。此定义创建了一个允许对「pods」执行「get」、「watch」和「list」操作的角色。

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
  - apiGroups: [""] # "" 表示核心 API 组
    resources: ["pods"]
    verbs: ["get", "watch", "list"]
```

保存并退出（`Ctrl+X`，`Y`，`Enter`）。

现在，将此卡片应用到集群中：

```bash
kubectl apply -f role.yaml
```

验证它是否存在：

```bash
kubectl get roles
```

你应该能看到 `pod-reader`。
