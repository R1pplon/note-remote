# 简介

Kubernetes 中的授权通常通过 **RBAC**（**基于角色的访问控制**，Role-Based Access Control）来处理。

- **用户 (User)**：员工（例如「Jane」）。
- **角色 (Role)**：带有特定访问权限的「门禁卡」（例如「可以进入服务器机房」、「可以阅读文档」）。
- **角色绑定 (RoleBinding)**：将「门禁卡」交给员工的行为。

RBAC 至关重要，因为集群访问不仅仅涉及身份验证。在确认身份后，系统仍需决定该身份被允许执行哪些操作。

在本实验中，你将担任安全主管：

- 创建一个「访客通行证」（Role），允许查看事物但不能进行修改。
- 将此通行证分配给名为 `jane` 的用户。
- 验证 Jane 是否可以查看集群，并在她尝试进行任何更改时被拒绝。
