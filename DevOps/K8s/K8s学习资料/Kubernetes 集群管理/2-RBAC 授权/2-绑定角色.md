# 绑定角色

现在我们有了卡片（`pod-reader`）和员工（`jane`）。但 Jane 还没有这张卡。我们需要一个 **RoleBinding** 来将它们关联起来。

`Role` 本身不会授予任何权限，除非通过绑定进行引用。这种分离是有意为之的。它允许管理员定义可重用的权限包，并根据需要将其附加到不同的用户、组或服务账户上。

创建一个名为 `rolebinding.yaml` 的文件：

```bash
nano rolebinding.yaml
```

粘贴以下内容。这表示：「将 `pod-reader` 角色绑定到用户 `jane`」。

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
  namespace: default
subjects:
  - kind: User
    name: jane # 员工
    apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader # 门禁卡
  apiGroup: rbac.authorization.k8s.io
```

保存并退出（`Ctrl+X`，`Y`，`Enter`）。

应用绑定：

```bash
kubectl apply -f rolebinding.yaml
```

现在，你已正式授予 Jane 对 default 命名空间的只读访问权限。
