# 验证限制

安全性也关乎你「不能」做什么。由于 Jane 只有 `pod-reader` 角色，她应该被严格禁止进行任何更改。

这就是 **最小权限原则** 的实际应用。与其授予广泛的访问权限并寄希望于人们谨慎行事，不如从任务所需的最小权限开始，仅在有实际操作需求时才进行扩展。

检查 Jane 是否可以创建 Pod：

```bash
kubectl auth can-i create pods --as jane
```

输出应该是 `no`。

检查 Jane 是否可以删除 Pod：

```bash
kubectl auth can-i delete pods --as jane
```

输出应该是 `no`。

如果 Jane 尝试真正执行这些命令，API 服务器会以「Forbidden」（禁止）错误拒绝她的请求。这证实了你的集群是安全的，并遵循了 **最小权限原则**。
