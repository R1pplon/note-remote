# 简介

部署工作并不总是按计划进行。当出现故障时，你需要一套可重复的排查方法。

在本实验中，你将解决**「缺失镜像案」**。

在 Kubernetes 中，排查工作通常从查看对象状态开始，接着是检查事件，最后进行更详细的分析。本实验重点介绍这种基于证据的排查工作流。

你将完成以下任务：

- **制造案发现场**：故意部署一个损坏的 Pod。
- **调查取证**：使用「望远镜」(`get`) 发现异常。
- **收集证据**：使用「放大镜」(`describe`) 找到根本原因。
- **破案**：修复配置并观察应用程序恢复正常。
