# 观察故障

现在，让我们查看部署的状态。这就像使用望远镜观察现场。

`ErrImagePull` 和 `ImagePullBackOff` 等状态只是症状，而非最终解释。它们告诉你下一步该去哪里调查。优秀的排查人员会将状态输出视为第一条线索，而不是最终诊断结果。

检查默认命名空间中的 Pod：

```bash
kubectl get pods
```

你可能首先会看到 `ContainerCreating`，但几秒钟后，它很可能会变为 **`ImagePullBackOff`** 或 **`ErrImagePull`**。

**ImagePullBackOff** 是 Kubernetes 的术语，意思是：「我尝试拉取此镜像，但失败了。我将退避（等待）一会儿，然后再试一次。」

这证实了受害者（Pod）处于不健康状态。
