# 应用资源限制

现在我们需要制定规则。我们不希望 **dev** 团队耗尽整个集群的 CPU 和内存。

我们将创建一个 `ResourceQuota` 对象。这有助于我们规定：「在 **dev** 命名空间中，最多只能使用 2 个 CPU 和 2Gi 内存。」

在共享集群中，配额尤为重要，因为 CPU 和内存是有限的。如果没有配额，一个嘈杂的工作负载可能会消耗超过其应得的份额，导致集群对其他人来说变得不可靠。从这个意义上说，配额不仅是关于核算，也是一种稳定性工具。

创建一个名为 `quota.yaml` 的文件：

```bash
nano quota.yaml
```

粘贴以下配置：

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: dev-quota
  namespace: dev
spec:
  hard:
    requests.cpu: "1"
    requests.memory: 1Gi
    limits.cpu: "2"
    limits.memory: 2Gi
```

保存并退出（`Ctrl+X`，`Y`，`Enter`）。

将此规则应用到集群：

```bash
kubectl apply -f quota.yaml
```

你可以查看规则详情。它会显示「硬性」（Hard）限制（你设置的值）和「已使用」（Used）数量（目前为 0）。

```bash
kubectl describe resourcequota dev-quota --namespace=dev
```
