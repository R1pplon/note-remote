# 创建命名空间

在这一步中，你将创建一个新的命名空间，为开发团队隔离资源。

命名空间是集群内部的一个逻辑边界，而不是一个独立的集群。团队仍然共享同一个控制平面和底层节点，但名称、策略和许多资源视图可以被分开，从而使工作保持井然有序。

默认情况下，所有内容都会进入 `default` 命名空间（即大厅）。让我们看看当前存在哪些命名空间：

```bash
kubectl get namespaces
```

你应该能看到 `default`、`kube-system`（楼宇管理员所在的地方）以及其他几个命名空间。

现在，让我们建造 `dev` 公寓：

```bash
kubectl create namespace dev
```

验证它是否已创建：

```bash
kubectl get namespaces
```

你会在列表中看到 `dev`。它目前是一个空公寓，没有墙壁或限制——你可以把任何东西放进去。
