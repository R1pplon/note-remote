# 简介

Kubernetes 集群就像一栋**大型公寓楼**。

如果所有人都住在大厅里，那将会非常混乱。你不知道谁的家具是谁的，而且一个吵闹的邻居可能会打扰到所有人。

**命名空间（Namespaces）** 在逻辑上对集群进行分区，以便不同的团队、环境或应用程序可以分别组织资源。

**资源配额（ResourceQuotas）** 则限制了命名空间可以消耗的资源总量。这有助于防止某个团队或工作负载耗尽共享的集群容量。

在本实验中，你将：

- 建造一个新的「公寓」（命名空间），命名为 `dev`。
- 为该公寓设置严格的「用电限额」（资源配额）。
- 尝试将家具（Pod）搬进公寓，并观察它是否符合限额要求。
