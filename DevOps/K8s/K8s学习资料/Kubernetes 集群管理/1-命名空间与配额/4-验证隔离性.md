# 验证隔离性

最后，让我们验证一下我们的「公寓」墙壁是否真实有效。

命名空间的隔离主要体现在名称、策略和操作范围上。默认情况下，这并不意味着工作负载运行在物理上独立的硬件上。当你后续学习节点选择和更强的隔离机制时，这种区别会变得很重要。

我们将 `dev-pod` 放在了 `dev` 中。它**不应该**在大厅（`default`）中可见。

列出默认命名空间中的 Pod：

```bash
kubectl get pods
```

你应该会看到「No resources found in default namespace」（或者至少看不到 `dev-pod`）。

现在，检查默认命名空间中的配额：

```bash
kubectl get resourcequota
```

你应该什么也看不到。我们设置的用电限额**仅**适用于 `dev` 公寓，而不是整栋大楼。

这证实了命名空间确实为你的团队提供了真正的隔离。
