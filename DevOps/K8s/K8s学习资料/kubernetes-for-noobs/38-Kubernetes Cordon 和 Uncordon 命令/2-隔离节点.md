# 隔离节点

在这一步骤中，我们将使用 `cordon` 命令将节点标记为 "unschedulable"（不可调度），从而阻止新的 Pod 被调度到该节点上。以下是具体步骤：

1. 使用以下命令列出集群中的节点：

```bash
kubectl get nodes
```

2. 使用以下命令隔离节点：

```bash
kubectl cordon minikube
```

3. 使用以下命令检查节点的状态，验证节点是否已被隔离，查看 `SchedulingDisabled` 字段：

```bash
kubectl get node | grep SchedulingDisabled
```
